# PRD: v2.0「角色生态」

> Generated: 2026-05-27 14:20:00
> Type: Feature | Priority: P1 | Est: 16h

## 1. 目标用户/场景

**目标用户**：希望分享角色、备份数据到云端、跨设备同步的用户
**使用场景**：
- 用户将精心打造的角色分享给朋友
- 用户换手机后云端恢复所有数据
- 用户在多个设备上使用同一批角色

**用户痛点**：
- 角色只能手动导出 JSON，无法快速分享
- 没有云端备份，重装后数据全部丢失
- 多设备使用时数据不同步

## 2. 功能列表

### MVP 功能

#### 角色 JSON 导出/导入
- [ ] 角色列表长按菜单添加「导出」选项
- [ ] 导出单个角色为 JSON 文件（含头像 Base64、描述、标签、开场白、记忆开关）
- [ ] 角色详情页独立「导出」入口
- [ ] 从文件管理器选择 JSON 文件导入角色
- [ ] 导入时检测重复（覆盖/新建/取消）
- [ ] 导出支持分享到其他应用

#### 角色二维码分享
- [ ] 生成包含角色信息的二维码（编码角色 JSON）
- [ ] 二维码扫描入口（从分享链接/相册扫码）
- [ ] 扫码后解析并预览角色信息
- [ ] 确认后导入角色
- [ ] 支持彩色/高密度二维码（ZXing）

#### 云端备份同步（可选）
- [ ] 用户自行注册/登录账号（邮箱+密码，无需手机号）
- [ ] 手动触发云端备份（上传角色数据、设置、记忆）
- [ ] 手动触发云端恢复（下载并覆盖本地数据）
- [ ] 备份前显示预览（角色数、数据大小）
- [ ] 支持自动备份开关（Wi-Fi 下自动备份）
- [ ] 多设备同步：登录同一账号自动拉取最新数据
- [ ] 冲突处理：本地优先或云端优先（用户选择）

### 扩展功能
- [ ] 角色社区：热门角色排行榜（需服务端）
- [ ] 角色市场：浏览并下载其他用户分享的角色（需服务端）

## 3. 数据模型

| 实体 | 字段 | 说明 |
|------|------|------|
| CloudAccount | id, email, createdAt | 用户账号 |
| CloudBackup | id, accountId, createdAt, dataHash | 备份记录 |
| CharacterExport JSON | name, avatar(B64), description, tags, prompt, openingGreeting, enableLongTermMemory, exportedAt | 导出格式 |

## 4. API 接口草图

```
POST /api/account/register   账号注册
POST /api/account/login     账号登录
POST /api/backup/upload      上传备份（角色+记忆 ZIP）
GET  /api/backup/latest     获取最新备份
POST /api/backup/restore     恢复备份
GET  /api/backup/list       备份历史列表
```

**本地存储**（无需网络的功能）：
```kotlin
// 导出角色
val json = Gson().toJson(character.toExport())
shareFile(json, "character_${name}.json")

// 导入角色
val data = Gson().fromJson(json, CharacterExport::class.java)
val character = data.toCharacter()
characterRepository.insert(character)
```

## 5. 用户旅程

### 旅程1: 导出并分享角色
1. 用户在角色列表长按某角色
2. 选择「导出」
3. 系统生成 JSON 文件并弹出分享面板
4. 用户选择分享方式（微信/QQ/邮件等）
5. 接收方打开 JSON 文件即可导入

### 旅程2: 云端备份
1. 用户进入「设置」→「云端备份」
2. 用户注册/登录账号
3. 点击「立即备份」
4. 显示备份内容预览（角色数、是否含 API 配置）
5. 确认后上传
6. 备份成功，显示备份时间

### 旅程3: 多设备同步
1. 用户在新设备登录同一账号
2. 系统自动检测到云端有备份
3. 弹出「发现云端备份，是否恢复？」对话框
4. 用户选择恢复
5. 角色数据同步完成

## 6. 非功能需求

- **性能**：二维码生成 < 200ms，解析 < 500ms
- **兼容性**：Android 8.0+
- **存储**：JSON 单角色 < 100KB，二维码建议分辨率 512x512
- **安全性**：账号密码 SHA-256 哈希存储，HTTPS 传输备份数据
- **隐私**：云端存储用户自主选择，数据加密传输

## 7. 风险与边界

> ⚠️ Assumption：云端备份需自建服务端（初期可使用第三方如 Firebase Firestore）
> ⚠️ Assumption：二维码内容有限，超大角色描述可能超出二维码容量（建议限制描述长度）
> ⚠️ Assumption：多设备同步可能产生数据冲突，需要用户选择策略

**风险**：
- 自建服务端成本高，可先用 Firebase/Supabase 等 BaaS
- 二维码在小屏幕设备可能扫描困难
- 云端备份占用服务端存储，需考虑免费额度

## 8. MVP 范围与不做清单

### MVP 包含
- 角色 JSON 导出/导入
- 二维码分享（生成+扫码）
- 云端备份（手动）
- 多设备同步

### MVP 不包含
- 角色社区/市场
- 自动云端备份（Wi-Fi）
- 备份历史管理
- 增量备份
