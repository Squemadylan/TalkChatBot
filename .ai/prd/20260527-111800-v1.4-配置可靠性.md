# PRD: v1.4「配置可靠性」

> Generated: 2026-05-27 11:18:00
> Type: Feature | Priority: P1 | Est: 12h

## 1. 目标用户/场景

**目标用户**: 新用户首次配置 API，老用户排查配置问题
**使用场景**:
- 新用户注册硅基流动后，一键填入推荐配置
- 用户配置错误时，能明确知道问题原因
- 用户能看到当前配置的完整摘要

**用户痛点**:
- 配置错误时提示不够明确（Key 错、URL 错、余额不足分不清）
- 没有模型预设，需要手动填写模型名
- 无法快速验证配置是否正确

## 2. 功能列表

### MVP 功能

#### API 一键检测
- [ ] 在配置页添加「检测连接」按钮
- [ ] 发送 test 请求验证 API Key + Base URL + 模型名
- [ ] 支持超时处理（10s）
- [ ] 显示检测结果（成功/失败及原因）

#### 错误提示优化
- [ ] 识别 HTTP 401: "API Key 无效"
- [ ] 识别 HTTP 403: "余额不足或无权限"
- [ ] 识别 HTTP 404: "模型不存在"
- [ ] 识别网络错误: "网络不可达"
- [ ] 识别超时: "请求超时"

#### 模型预设选择
- [ ] 在配置页添加模型预设下拉框
- [ ] 预设选项: DeepSeek V3.2, DeepSeek V3
- [ ] 选择后自动填入 Base URL 和模型名
- [ ] 保留自定义输入选项

#### 配置摘要
- [ ] 显示当前 API Base URL（脱敏显示）
- [ ] 显示当前模型名
- [ ] 显示最后检测时间
- [ ] 一键复制当前配置

### 扩展功能
- [ ] 支持多套 API 配置切换
- [ ] 配置导入/导出

## 5. 用户旅程

### 旅程1: 新用户一键配置
1. 用户进入「配置」页
2. 用户点击「选择模型预设」
3. 用户选择「DeepSeek V3.2」
4. Base URL 和模型名自动填入
5. 用户填入 API Key
6. 用户点击「检测连接」
7. 系统显示「配置正确」

### 旅程2: 排查配置错误
1. 用户填入 API 信息后点击「检测」
2. 系统返回「401: API Key 无效」
3. 用户检查并修正 Key
4. 再次检测通过

## 6. 非功能需求

- **性能**: 检测请求 10s 内返回结果
- **兼容性**: Android 8.0+
- **容错**: 网络不可用时显示友好提示

## 7. 风险与边界

> ⚠️ Assumption: 检测会消耗少量 API 调用
> ⚠️ Assumption: 不同 API 提供商返回的错误码格式可能不同
