# PRD: v1.9「隐私与安全」

> Generated: 2026-05-27 14:20:00
> Type: Feature | Priority: P2 | Est: 12h

## 1. 目标用户/场景

**目标用户**：注重隐私的用户，有敏感角色需要保护的用户
**使用场景**：
- 用户不希望他人随意打开应用查看聊天记录
- 用户有专属私密角色不希望被他人看到
- 用户想控制通知栏是否显示消息内容

**用户痛点**：
- 应用无任何隐私保护，任何人都能查看聊天
- 通知栏默认显示消息内容，隐私泄露风险
- 共享设备时无法保护特定角色

## 3. 数据模型

| 实体 | 字段 | 说明 |
|------|------|------|
| SharedPreferences | app_lock_enabled | 0=关闭, 1=PIN, 2=生物识别 |
| SharedPreferences | app_lock_pin | PIN 码 SHA-256 哈希 |
| SharedPreferences | app_lock_timeout | 超时毫秒数 |
| SharedPreferences | notification_show_preview | 0=关闭, 1=开启 |
| SharedPreferences | global_notification_enabled | 全局通知开关 |
| Character | isLocked | 是否加锁 |
| Character | notificationEnabled | 单角色通知开关 |

## 4. 用户旅程

### 旅程1: 设置应用锁
1. 用户进入「设置」页
2. 点击「应用锁」
3. 选择解锁方式（PIN / 指纹）
4. 如选 PIN，输入 4-6 位数字并确认
5. 设置锁定超时时间
6. 开启成功，退出会话需解锁

### 旅程2: 加锁私密角色
1. 用户在角色列表页编辑某角色
2. 开启「加锁」开关
3. 保存
4. 该角色在列表显示锁图标
5. 其他用户无法直接进入对话

### 旅程3: 收到通知
1. 角色收到新消息
2. 系统发送通知
3. 若通知预览开启，显示消息摘要
4. 若关闭，只显示应用名
5. 点击通知打开应用（需解锁）

## 5. 非功能需求

- **性能**：解锁操作 < 500ms
- **兼容性**：Android 8.0+，指纹需 Android 9+
- **安全性**：PIN 码本地 SHA-256 存储，不明文存储
- **容错**：无指纹设备时隐藏指纹选项

## 6. 风险与边界

> ⚠️ Assumption：生物识别依赖系统 BiometricPrompt，部分设备可能不支持人脸
> ⚠️ Assumption：应用被强制停止时锁仍然有效（进程级别）
> ⚠️ Assumption：ROOT 设备可能绕过应用锁（免责声明）

**风险**：
- 无指纹设备用户只能使用 PIN
- 部分设备生物识别安全性参差不齐
