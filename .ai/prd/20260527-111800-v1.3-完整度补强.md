## 1. 目标用户/场景

**目标用户**: 现有活跃用户，对应用有一定了解
**使用场景**:
- 用户在设置页调整气泡样式和回复策略，获得更好的视觉体验
- 新用户首次配置 API 时，能快速验证配置正确性
- 用户在设置页看到的功能都是可用的，没有"占位"功能

**用户痛点**:
- 气泡样式、回复策略等设置项虽定义但未实现
- 设置页存在不可用的"配图/语音"入口
- API 配置错误时提示不够明确

## 3. 数据模型

| 实体 | 字段 | 说明 |
|------|------|------|
| SharedPreferences | chat_bubble_style | 0=默认, 1=紧凑, 2=圆角, 3=半透明 |
| SharedPreferences | reply_style | 0=标准, 1=短回复, 2=细腻 |
| SharedPreferences | status_bar_immersive | 0=关闭, 1=开启 |
| 无新增数据库表 | | |

## 4. 接口草图

本版本无新增 API 接口，所有配置通过 SharedPreferences 存储：

```kotlin
// 气泡样式
getSharedPreferences(PREFS_NAME).getInt(KEY_CHAT_BUBBLE_STYLE, 0)

// 回复策略
getSharedPreferences(PREFS_NAME).getInt(KEY_REPLY_STYLE, 0)

// 状态栏沉浸
getSharedPreferences(PREFS_NAME).getInt(KEY_STATUS_BAR_IMMERSIVE, 0)
```

## 5. 用户旅程

### 旅程1: 调整气泡样式
1. 用户进入「设置」页
2. 用户点击「气泡样式」
3. 弹出选择对话框（默认/紧凑/圆角/半透明）
4. 用户选择样式
5. 返回聊天页，样式已更新

### 旅程2: 验证API配置
1. 用户进入「配置」页
2. 用户填写 API 信息
3. 用户点击「检测连接」
4. 系统显示检测结果（成功/Key错误/余额不足等）

## 6. 非功能需求

- **性能**: UI 切换 100ms 内完成
- **兼容性**: Android 8.0+ (API 26)

- **安全性**: 不上传任何用户数据

## 7. 风险与边界

> ⚠️ Assumption: 气泡样式修改仅影响当前会话，退出后保持
> ⚠️ Assumption: 回复策略修改影响后续所有请求
> ⚠️ Assumption: 状态栏沉浸式在 Android 10+ 效果最佳

**风险**:
- 半透明气泡需要处理深色/浅色模式适配
- 不同屏幕密度下圆角效果可能不一致
