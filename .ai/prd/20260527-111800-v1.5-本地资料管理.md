## 1. 目标用户/场景

**目标用户**: 需要迁移角色数据的用户，需要备份/恢复配置的用户
**使用场景**:
- 用户换手机时，导出角色数据到新手机
- 用户想要分享角色卡给其他用户
- 用户备份时不想包含敏感的 API Key

## 4. 接口草图

```kotlin
// 导出角色
val character = characterRepository.getById(id)
val json = Gson().toJson(character.toExport())
shareFile(json, "character_${name}.json")

// 导入角色
val importData = Gson().fromJson(json, CharacterExport::class.java)
val character = importData.toCharacter()
characterRepository.insert(character)
```

## 5. 用户旅程

### 旅程1: 导出角色
1. 用户在角色列表长按某角色
2. 选择「导出」
3. 系统生成 JSON 文件
4. 系统弹出分享对话框
5. 用户选择分享方式

### 旅程2: 备份恢复
1. 用户进入设置页
2. 点击「恢复备份」
3. 选择备份文件
4. 系统显示预览（角色数、是否含配置）
5. 用户确认恢复
6. 系统执行恢复操作

## 6. 非功能需求

- **文件大小**: 单角色 JSON < 100KB
- **兼容性**: 导出 JSON 格式保持向后兼容

## 7. 风险与边界

> ⚠️ Assumption: 头像文件以 Base64 嵌入 JSON 或外部引用
> ⚠️ Assumption: 导入时同名角色默认创建新角色而非覆盖
